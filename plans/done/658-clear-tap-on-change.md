# Plan: Clear tap number when selecting brew/location/brewtype (issue #658)

## Decisions

- Clearing is done client-side in JS, not server-side, for immediate UX feedback.
- The tap field is cleared whenever any of these three change: Brew, Location, or BrewType.
- Clearing applies in both new-glass and edit modes (if you change the brew or location, the tap number no longer makes sense).
- The `data-rawval` attribute on the tap input is **not** cleared — it holds the persisted value and is only used by `editrecord()` to restore the original value when switching to edit mode.
- Server-side validation (checking tap against `tap_beers` table in `postglass.pm`) is out of scope for now; the issue mentions it as a "maybe".

## No database changes

No schema changes required.

## Phases

### Phase 1 — Clear tap when BrewType changes (`code/glasses.pm`)

In `selectbrewtype()`, the inline `selbrewchange()` JS function already handles BrewType changes.
Add a line at the top of `selbrewchange()` to clear the tap input:

```javascript
function selbrewchange(el) {
  // Clear tap number — brew type changed
  const tapInput = document.querySelector('[name=tap]');
  if (tapInput) tapInput.value = '';
  ...existing code...
}
```

File: `code/glasses.pm`, inside `selectbrewtype()`, in the inline `<script>` block (~line 77).

### Phase 2 — Clear tap when Brew or Location changes (`code/glasses.pm`)

In `maininputform()`, in the deferred `<script>` block at the bottom (~line 320), add event
listeners on the hidden inputs for the Brew and Location custom dropdowns.

The hidden inputs are `id='Brew'` and `id='Location'` (generated by `inputs::dropdown()`).
Both already dispatch a `new Event('input')` on the hidden input when an item is selected
(see `static/inputs.js`, item click handler, ~line 228).

Add to the `$script` heredoc in `maininputform()`:

```javascript
// Clear tap when brew or location changes
['Brew', 'Location'].forEach(function(name) {
  const hidden = document.getElementById(name);
  if (hidden) {
    hidden.addEventListener('input', function() {
      const tapInput = document.querySelector('[name=tap]');
      if (tapInput) tapInput.value = '';
    });
  }
});
```

File: `code/glasses.pm`, inside `maininputform()`, in the `$script` heredoc (~line 295).

## Open questions

- Should the tap also be cleared when the user _clears_ the Brew or Location field (sets it back to blank), or only when a real item is selected? Currently `dispatchEvent` fires on any hidden input change including clearing.
- Is the server-side check (`postglass.pm` validating tap against `tap_beers`) wanted for a follow-up issue, or should it be added here too?
